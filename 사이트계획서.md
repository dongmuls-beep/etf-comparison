ETF 실부담비용 자동 비교 웹사이트
Phase 1: 구글 시트 세팅 (관리자 페이지 만들기)
•	목표: 구글 스프레드시트를 '데이터베이스' 겸 '종목 관리소'로 만들기
•	실행 내용: 하나의 구글 시트 안에 두 개의 탭을 만듭니다.
1.	종목관리 시트: 사용자가 비교하고 싶은 ETF 종목명(예: TIGER 미국S&P500 등)을 자유롭게 적어두는 곳입니다. 구분/종목코드/종목명으로 기록합니다. 구분이 일치하는 항목끼리 수수료를 비교하게 됩니다.
2.	수수료결과 시트: 파이썬이 데이터를 정리해서 최종 결과를 뿌려줄 빈 공간입니다.
•	사용자님의 역할: 구글 시트 파일을 하나 만들고, 탭 두 개를 생성한 뒤 종목관리 탭에 당장 비교하고 싶은 ETF 이름들을 쭉 적어주시면 됩니다.
Phase 2: 데이터 수집 및 스마트 필터링 (파이썬 요리사)
•	목표: 금융투자협회 엑셀을 다운로드하고, 종목관리 시트의 리스트와 대조하여 알맹이만 뽑아내기
•	실행 내용: 파이썬 코드가 구글 시트의 종목관리 탭을 먼저 읽어옵니다. "아, 오늘은 이 15개 종목을 비교하시는군요!"라고 파악한 뒤, 협회 엑셀을 다운로드합니다. 
-	📊 금융투자협회 엑셀 파일 다운로드 4단계
-	공시 사이트 접속하기
-	인터넷 브라우저를 열고 금융투자협회 전자공시서비스(dis.kofia.or.kr) 에 접속합니다.
-	해당 메뉴 찾아가기
-	사이트 상단의 메인 메뉴에서 **[펀드공시]**에 마우스를 올립니다.
-	아래로 펼쳐지는 하위 메뉴 중에서 **[펀드보수 및 비용비교]**를 클릭합니다. (세부 메뉴로 '펀드별 보수비용 비교'를 선택하시면 됩니다.)
-	검색 조건 세팅하기
-	화면 상단에 조회하고자 하는 조건을 넣는 칸이 나옵니다. 기준일(보통 가장 최근 달의 말일)이 기본으로 설정되어 있을 겁니다.
-	우리는 나중에 파이썬으로 데이터를 걸러낼 것이기 때문에, 빈칸에 특정 ETF 이름을 적지 말고 전체 조건 그대로 [검색] 버튼을 눌러줍니다.
-	엑셀(Excel) 다운로드하기
-	검색 결과로 수만 개의 펀드 목록이 화면에 나타납니다.
-	표의 오른쪽 위나 아래쪽을 보시면 [엑셀다운로드] 버튼 또는 [Excel] 모양의 아이콘이 있습니다.
-	이 버튼을 누르시면 전체 펀드의 보수 데이터가 담긴 엑셀 파일이 컴퓨터에 저장됩니다!
-	
•	그안의 파일 수만 개의 데이터 중에서 딱 그 15개 종목의 '실질부담비용' 관련 수치만 쏙쏙 골라냅니다.
Phase 3: 구글 시트 자동 적재 (비서 역할)
•	목표: 정제된 데이터를 수수료결과 시트에 예쁘게 덮어쓰기
•	실행 내용: 2단계에서 골라낸 데이터를 수수료결과 탭의 A1 칸부터 깔끔하게 표 형태로 자동 입력합니다. 기존 데이터는 지우고 항상 최신 데이터로 업데이트되게 만듭니다.
Phase 4: 정적 HTML 프론트엔드 화면 구현 (웹사이트 완성)
•	목표: 구글 시트에 쌓인 데이터를 깔끔한 웹사이트 화면으로 띄워주기
•	실행 내용: Google Apps Script(GAS)를 활용해 수수료결과 시트의 데이터를 바깥으로 내보내는 API를 간단하게 만들고, 이를 정적 HTML 프론트엔드 화면에서 호출하여 방문자들에게 시각적인 비교 표로 보여줍니다. 별도의 복잡한 서버를 구축할 필요 없이 무표로 빠르고 가볍게 사이트를 돌릴 수 있는 최고의 방법입니다.
Phase 5: 완전 자동화 세팅 (로봇 스케줄러)
•	목표: 매달 내가 신경 쓰지 않아도 최신 엑셀이 웹사이트까지 알아서 반영되게 만들기
•	실행 내용: 무료 서비스인 'GitHub Actions'를 이용해 매월 특정 날짜가 되면 파이썬 코드가 알아서 돌도록 세팅합니다.
